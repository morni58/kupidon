ЛЬТИМАТИВНЫЙ PRODUCT REQUIREMENTS DOCUMENT (PRD)

Уровень: Субатомная детализация алгоритмов и функций

Данный документ описывает исчерпывающую логику работы Telegram Mini App (CupidBot). Описан каждый клик, валидация, реакция интерфейса и запросы к базе данных.

1. БАЗОВАЯ АРХИТЕКТУРА И UI/UX ДВИЖОК

Технологический процесс: При нажатии кнопки "Найти пару" в боте, Telegram открывает Web App. Фронтенд (React) сразу запрашивает у Telegram window.Telegram.WebApp.initDataUnsafe. Полученный user.id и hash отправляются на бэкенд (FastAPI) для авторизации (JWT-токен).

UI/UX паттерны: * Приложение занимает 100% высоты экрана (без скролла на главном экране).

Используется Glassmorphism: панели имеют полупрозрачный белый/темный фон с backdrop-filter: blur(12px).

Виброотклик: Каждое нажатие кнопки, свайп или появление модалки вызывает функцию TWA SDK: HapticFeedback.impactOccurred('light' | 'medium' | 'heavy').

2. АЛГОРИТМ ОНБОРДИНГА И РЕДАКТИРОВАНИЯ ПРОФИЛЯ

Функция запускается, если юзер заходит впервые (нет записи в БД) или нажимает "Редактировать" в Профиле.

Экран 1: Базовые данные (Имя и Возраст)

UI: Поле ввода имени. Барабан (Date Picker) для выбора даты рождения.

Логика/Валидация: Фронтенд не дает ввести спецсимволы в имя. При выборе даты рождения фронтенд вычисляет возраст.

Ограничение: Если возраст < 18, кнопка "Далее" блокируется, появляется красная надпись: "Доступ разрешен только с 18 лет". На бэкенд отправляется дата рождения, а не сам возраст (чтобы возраст обновлялся каждый год автоматически).

Экран 2: Поиск (Гендерная фильтрация)

UI: Две группы кнопок. "Я: [Парень] [Девушка]". "Ищу: [Парня] [Девушку] [Всех]".

Логика: Эти данные формируют жесткие фильтры gender и search_gender в SQL-запросах для ленты.

Экран 3: Локация (Smart Geo)

UI: Кнопка "Найти меня (GPS)" и строка поиска города.

Логика: При нажатии GPS запрашивается разрешение браузера. Координаты отправляются на бэкенд, который через гео-библиотеку (или PostGIS) находит ближайший город из нашей базы и привязывает юзера к нему.

Экран 4: Медиа-слоты (Экономика контента)

UI: Сетка из 5 пустых квадратов со знаком "+".

Логика: При загрузке файла бэкенд проверяет MIME-тип.

Если это фото (до 5 МБ) — оно сжимается (WebP) и занимает 1 слот (в БД создается 1 запись Media_Slots с slot_index).

Если это видео (до 10 сек, до 15 МБ) — фронтенд визуально объединяет 3 квадрата в один большой прямоугольник. Бэкенд проверяет, есть ли 3 свободных слота. Если нет — ошибка: "Освободите 3 слота для видео".

Экран 5: Динамические Теги

UI: Облако кнопок с интересами. Юзер может выбрать до 5 штук.

Логика: Фронтенд запрашивает GET /api/tags и рендерит их с нужными HEX-цветами обводки. Выбранные ID тегов сохраняются в связную таблицу User_Tags.

3. ДВИЖОК СВАЙПОВ И АЛГОРИТМ ВЫДАЧИ (MATCHING ENGINE)

Функция генерации ленты (GET /api/feed):
Бэкенд генерирует пачку из 10-15 анкет, чтобы юзер не ждал загрузки при каждом свайпе.

Фильтры SQL: 1. Исключить user_id, которые уже есть в таблице Swipes (где actor_id = текущий юзер).
2. Пол совпадает с search_gender юзера.
3. Гео: город совпадает, либо находится в радиусе (Smart Geo fallback).
4. Исключить тех, у кого is_stealth_mode = True (Олигархи).
5. Если юзер в 18+ режиме, искать ТОЛЬКО тех, у кого is_18_mode_active = True.

Сортировка SQL (ORDER BY): 1. tier = 'kupidon' (Абсолютный топ).
2. tier = 'premium' (Буст).
3. created_at > now() - 2 hours (Новички).
4. Обычные пользователи.

Функция: Свайп Влево (Пропуск)

Действие: Юзер смахивает карточку влево. Карточка улетает за экран.

Бэкенд: Отправляется POST /api/swipe с type: "left". Создается запись в БД Swipes. Списывается 1 лимит swipes_left.

Функция: Свайп Вправо (Лайк)

Действие: Юзер смахивает вправо.

Бэкенд: POST /api/swipe с type: "right". Списывается 1 свайп.

Проверка Мэтча: Бэкенд делает запрос к таблице Swipes: Лайкал ли меня этот юзер ранее? * Если НЕТ -> конец функции.

Если ДА -> Создается запись в таблице Matches. Бэкенд возвращает флаг is_match: true.

Реакция UI: Фронтенд стопит свайпы и показывает на весь экран модалку "У ВАС СОВПАДЕНИЕ!" с аватарками обоих и кнопкой "Написать".

Функция: Шаг Назад (⏪ Rewind)

Действие: Клик по желтой стрелке.

Бэкенд: POST /api/rewind.

Если tier == free, бэкенд возвращает статус 403. Фронтенд показывает Paywall (окно покупки Premium).

Если tier == premium/kupidon, бэкенд находит последнюю запись swipe_left за последние 24 часа, удаляет ее, восстанавливает 1 лимит свайпа и возвращает данные карточки обратно. Фронтенд анимированно возвращает карточку в центр экрана.

Функция: Пейволл (Окончание лимитов)

Действие: Бэкенд возвращает ошибку "Limit Exceeded", либо фронтенд видит, что swipes_left == 0.

Реакция UI: Компонент свайпов блокируется и покрывается Blur-фильтром. Вычисляется время до конца суток (00:00 МСК). Рендерится таймер обратного отсчета: ⏳ 04:12:59. Ниже кнопка "Снять ограничения (Купить Kupidon)".

4. ЭКОСИСТЕМА "СИМПАТИЙ" И ВРЫВОВ (FORCE CHAT)

Вкладка ❤️ Симпатии состоит из 3 секций.

Секция "Тебя лайкнули" (Ключевая монетизация):

Логика загрузки: GET /api/sympathies. Бэкенд отдает список юзеров, которые сделали свайп вправо на текущего юзера, но с которыми еще нет Matches.

UI: Фотографии рендерятся в полном качестве (без размытия).

Функция: Клик по анкете (Force Chat)

Юзер хочет написать тому, кто его лайкнул, НЕ дожидаясь, пока эта анкета попадется ему в общей ленте. Он кликает по фото.

Вылезает поп-ап проверки лимитов:

Если Free: "Чтобы ворваться в диалог прямо сейчас, заплатите 50 Stars". Кнопка [Оплатить].

Если Premium/Kupidon: "У вас осталось {X} врывов на сегодня. Начать чат?".

Бэкенд POST /api/force_chat: Если лимиты есть (или прошла оплата), бэкенд списывает 1 врыв (force_chats_used + 1) и принудительно создает запись в таблице Matches с флагом is_force_chat = true. Юзер мгновенно перебрасывается во вкладку "Чаты" в открытый диалог.

5. МЕССЕНДЖЕР И СИСТЕМА СОГЛАСИЯ (SAFE CHAT)

Чат реализован через WebSockets (или Long Polling) для мгновенной доставки сообщений.

Логика блокировки контактов (Telegram Username):

В верхней панели чата находится кнопка с иконкой Telegram. По умолчанию она серая (disabled).

Под ней висит текст: "Откроется через {N} сообщений".

Функция счетчика сообщений: При каждом отправленном сообщении бэкенд увеличивает messages_count в таблице Matches. Фронтенд динамически меняет цифру N.

Как только счетчик достигает порога (15 для Free, 5 для Premium), кнопка становится синей (активной). При клике копируется/открывается юзернейм собеседника.

Механика Мгновенного Согласия (Кнопка "Запросить ТГ"):

Внутри инпутов чата есть кнопка "Запросить ТГ".

Действие юзера 1: Нажимает кнопку. Отправляется POST /api/chat/request_tg.

Реакция UI для юзера 2: В чате рендерится системное сообщение (компонент ConsentMessageBubble): "Собеседник хочет перейти в Telegram". Две кнопки: [Отклонить] и [Одобрить].

Действие юзера 2: Если нажимает [Одобрить], отправляется POST /api/chat/approve_tg.

Результат: Бэкенд меняет флаги tg_unlocked_user1 = true и tg_unlocked_user2 = true. Блокировка мгновенно снимается для обоих, лимит сообщений аннулируется.

Ледоколы (Icebreakers):

Если messages_count == 0, над клавиатурой рендерятся 3 кнопки (например: "Собаки или кошки?", "Какая последняя песня в плейлисте?"). При клике текст автоматически отправляется как обычное сообщение.

6. СЕКРЕТНАЯ КОМНАТА (🔞 РЕЖИМ 18+)

Функция полной изоляции трафика.

Функция: Активация режима

В настройках профиля переключается тумблер is_18_mode_active = True.

UI: Глобальная цветовая тема приложения (через React Context) переключается с белой на Темно-бордовую/Черную.

Функция: Маршрутизация свайпов

Теперь алгоритм выдачи GET /api/feed на бэкенде жестко фильтрует базу: выдаются только те, у кого этот тумблер тоже включен. Для остальных юзеров в "белой" ленте этот профиль перестает существовать.

Функция: Принудительная Приватность в Чате

Если юзер переходит в чат, созданный в 18+ режиме (Matches.is_18_room == true), стандартная иконка прикрепления медиа (Скрепка) заменяется на иконку "Огонь".

Логика медиа: При отправке фото, фронтенд вешает на него флаг is_disappearing = true. В UI получателя фото отображается заблюренным. При клике оно открывается. Как только получатель закрывает фото, фронтенд отправляет POST /api/chat/burn_media, и файл навсегда удаляется с серверов (S3/Диска) и из интерфейса.

7. РЕЖИМ «ОЛИГАРХ» (ЭЛИТНЫЕ WHALE-ФУНКЦИИ)

Доступно только при tier == kupidon.

Функция: Стелс-Режим (Глухая невидимка)

Тумблер в настройках меняет is_stealth_mode = True в БД.

Алгоритм: Запрос выдачи ленты GET /api/feed у всех остальных пользователей добавляет условие WHERE is_stealth_mode = False. Таким образом VIP-юзера невозможно увидеть случайно.

Функция: Золотой Ключ (Прямой выкуп юзернейма)

UI: В режиме "Стелс" на карточках людей в ленте рядом с крестиком и лайком появляется золотая кнопка 🗝️.

Действие: VIP нажимает 🗝️.

Бэкенд: Вызов POST /api/buy_golden_contact.

Проверка баланса: stars_balance >= 1000.

Если ОК -> Списывается 1000 Stars.

Создается запись Matches в обход лайков. Флаги tg_unlocked_user1 и tg_unlocked_user2 сразу ставятся в True.

Адресату отправляется Telegram Push через бота: "Влиятельный VIP-пользователь выкупил ваш контакт за Stars!".

Бэкенд возвращает VIP-юзеру ссылку на профиль t.me/username.

8. ТАРИФНАЯ ЭКОНОМИКА (БИЗНЕС-ЛОГИКА)

FREE (База): 50 свайпов в сутки. (В 18+ режиме жесткий лимит — 5 свайпов). Возврата (⏪) нет. Открытие ТГ — 15 смс.

PREMIUM (Подписка): 200 свайпов в сутки. Буст sort_order в выдаче (анкеты показываются чаще). 3 бесплатных врыва (Force Chat) в сутки. Работает Возврат (⏪). Открытие ТГ — 5 смс.

KUPIDON (VIP Подписка): 500 свайпов в сутки. Абсолютный sort_order (первые в очереди). 15 врывов (Force Chat) в сутки. 5 суперлайков бесплатно. Доступ к "Стелсу" и выкупу "Золотого Ключа". ТГ-юзернейм открывается мгновенно (кнопка активна сразу, лимит 3 раза в день).

Telegram Stars (Расходники): Покупка Буста (поднятие в топ на 2 часа), Суперлайков, Врывов в чат и Золотых ключей. Бэкенд обрабатывает транзакции через sendInvoice Telegram API.

9. ПАНЕЛЬ GOD MODE (УПРАВЛЕНИЕ АДМИНИСТРАТОРОМ)

Реализовано в виде команд для бота (aiogram). Обрабатывается только если message.from_user.id in ADMIN_IDS. Никаких веб-админок.

Функция: /user {ID или @username}

Бот делает SELECT в БД, формирует текст с данными юзера (баланс, тариф, жалобы) и отправляет сообщение с Inline-клавиатурой:

[Выдать Купидон]: Callback-запрос, делает UPDATE tier = 'kupidon'. Бот пишет "Успешно".

[+Stars]: Бот переходит в состояние (FSM) ожидания суммы, после ввода делает stars_balance += N.

[18+ Вкл/Выкл]: UPDATE is_18_mode_active = NOT is_18_mode_active.

[Теневой Бан]: UPDATE is_shadowbanned = True.

Логика теневого бана: При GET /api/feed забаненному юзеру продолжают выдаваться анкеты (он думает, что всё работает). Но при POST /api/swipe его лайки физически не записываются в БД для других пользователей, и в ленте другим он не показывается (WHERE is_shadowbanned = False). Идеальная защита от спам-ботов.

Функция: /tags

Админ пишет: /tags Люблю Аниме | #FF0000 | 🎌.

Бот парсит строку по разделителю |.

Делает INSERT INTO Admin_Tags (name, color_hex, emoji). Хеш моментально доступен на фронтенде всем юзерам.

Функция: /economy

Позволяет налету делать UPDATE конфигурационной таблицы (например, уменьшить daily_free_swipes с 50 до 30 при нагрузке на сервера).

10. АРХИТЕКТУРА И ТИПИЗАЦИЯ БАЗЫ ДАННЫХ (PostgreSQL)

Таблица Users

id: UUID PRIMARY KEY

tg_id: BIGINT UNIQUE INDEX

name: VARCHAR(50)

birth_date: DATE (Возраст вычисляется динамически)

gender: ENUM('male', 'female')

search_gender: ENUM('male', 'female', 'any')

tier: ENUM('free', 'premium', 'kupidon') DEFAULT 'free'

swipes_left: INT DEFAULT 50 (Сброс по крону в 00:00 МСК)

force_chats_used: INT DEFAULT 0 (Сброс по крону)

stars_balance: INT DEFAULT 0

is_18_mode_active: BOOLEAN DEFAULT false

is_stealth_mode: BOOLEAN DEFAULT false

is_shadowbanned: BOOLEAN DEFAULT false

created_at: TIMESTAMP DEFAULT NOW()

Таблица Media_Slots

id: UUID PRIMARY KEY

user_id: UUID FOREIGN KEY (Users) ON DELETE CASCADE

media_url: VARCHAR (Ссылка на S3/CDN)

media_type: ENUM('photo', 'video')

slot_index: SMALLINT (Ограничение 1-5, UNIQUE вместе с user_id)

Таблица Matches

id: UUID PRIMARY KEY

user1_id: UUID FOREIGN KEY (Users)

user2_id: UUID FOREIGN KEY (Users)

is_force_chat: BOOLEAN DEFAULT false (Создано через Врыв)

is_18_room: BOOLEAN DEFAULT false (Создано в секретной комнате)

tg_unlocked_user1: BOOLEAN DEFAULT false (Дано согласие на ТГ)

tg_unlocked_user2: BOOLEAN DEFAULT false (Дано согласие на ТГ)

messages_count: INT DEFAULT 0

Таблица Swipes

id: UUID PRIMARY KEY

actor_id: UUID FOREIGN KEY (Кто свайпал)

target_id: UUID FOREIGN KEY (Кого свайпал)

action_type: ENUM('left', 'right', 'superlike')

created_at: TIMESTAMP DEFAULT NOW()

Индексы: Composite Index на (actor_id, target_id) для быстрого поиска дубликатов и взаимных лайков.

Таблица Admin_Tags

id: SERIAL PRIMARY KEY

name: VARCHAR(30)

color_hex: VARCHAR(7) (e.g. #FF00FF)

emoji: VARCHAR(10)

is_18_only: BOOLEAN DEFAULT false
