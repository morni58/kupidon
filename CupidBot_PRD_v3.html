2. Онбординг & Профиль

При нажатии «Найти пару» в боте Telegram открывает Web App. Фронт читает initDataUnsafe, отправляет на FastAPI hash+user.id → бэк валидирует подпись по HMAC от bot_token → выдаёт JWT. Онбординг запускается при первом входе (нет записи в БД).

1

Имя & Возраст

  • • Имя: буквы/пробелы/дефис, 2–30, trim(). Спецсимволы и цифры режутся в реальном времени.
  • • Барабан даты рождения (iOS-стиль), возраст считается на лету.
  • • < 18 → «Далее» disabled + красная надпись «Доступ с 18 лет».
  • • В БД пишется birth_date (DATE) — возраст пересчитывается каждый год.
2

Я / Ищу

  • • «Я»: [Парень] [Девушка] — обязательно.
  • • «Ищу»: [Парня] [Девушку] [Всех].
  • • Пишется в gender + search_gender, жёсткий WHERE в ленте.
3

Геолокация

  • • «Найти меня (GPS)» → POST /api/geo/resolve → PostGIS → ближайший город.
  • • Fallback: поиск города с автокомплитом.
  • • Smart Geo: анкеты кончились → радиус 50→100→200 км.
4

Медиа-слоты

  • • 5 слотов. Фото (до 5 МБ)=1 слот, WebP-сжатие. 1-е фото обязательно.
  • • Видео (до 10 сек, 15 МБ)=3 слота, UI сливает квадраты в прямоугольник.
  • • Видео при < 3 свободных слотах → «Освободите 3 слота».
  • • Каждый загруженный файл → AI-модерация (см. Trust & Safety).
5

Теги (до 5)

  • • GET /api/tags → облако кнопок с HEX-обводкой + эмодзи.
  • • Теги is_18_only видны только в 18+ режиме.
  • • 5-й выбран → остальные блокируются с анимацией «лимит».
  • • Совпадение тегов влияет на скоринг ленты (см. Мэтчинг).
6

О себе → Финал

  • • Textarea до 150 символов со счётчиком. Не обязательно, но +10% к «горячести».
  • • Автомодерация: стоп-слова, ссылки, контакты, регулярки на бэке.
  • • INSERT INTO users → предложение пройти верификацию → вход в ленту.
  • • Редактирование в любой момент из раздела Профиль.

3. Верификация new

Главное оружие против фейков и ботов. Дейтинг живёт на доверии: если женская аудитория верит, что анкеты настоящие — приложение растёт. Верификация добровольная, но даёт буст и галочку.

Селфи-жест (liveness)

Приложение показывает случайный жест (например, «коснись щеки», «повернись влево»). Фронт снимает короткое видео/3 кадра → POST /api/verify/selfie. На бэке: проверка живости + сравнение лица с фото профиля (face-embedding cosine ≥ порога). Совпало → is_verified=true, синяя галочка. Биометрия не хранится — только хэш-эмбеддинг для сверки, кадры удаляются после обработки.

Возрастной гейт для 18+

Вход в 18+ режим требует verified-статус + явное подтверждение совершеннолетия. birth_date уже в БД — повторно дата рождения не запрашивается. Это барьер от подростков, а не сбор документов.

Доверие Telegram-аккаунта

У аккаунта уже есть номер телефона в Telegram. Дополнительно проверяем «возраст» аккаунта и наличие username как мягкий сигнал доверия в анти-фрод скоринге. Свежесозданные пустые аккаунты получают пониженный trust_score.

Фильтр «Только Verified»

В настройках ленты — тумблер «Показывать только проверенных». Бесплатно для всех. Это превращает верификацию в сетевой эффект: проверишься сам → попадёшь в выдачу к тем, кто фильтрует.

4. Лента, Свайпы & Умный мэтчинг

Скоринг ленты (не только тариф) new

Релевантность важнее платности — иначе платники видят мусор, а лента деградирует. Итоговый порядок = взвешенная сумма:

  • • Tier-буст — kupidon > premium > free (вес умеренный, не доминирующий).
  • • Совпадение тегов — пересечение интересов.
  • • Свежесть/активность — last_active_at, новички < 48ч.
  • • Profile Score и verified-статус.
  • • Близость по гео (чем ближе — тем выше).
  • • Anti-staleness — лёгкий RANDOM(), чтобы лента не была одинаковой.

Кэш-пачка

GET /api/feed → 15 анкет, кэш в Redis (TTL 5 мин). Фронт хранит локально, дозапрашивает при ≤ 3 оставшихся. Нет ожидания на каждом свайпе.

Свайпы и матч

  • • Влево — POST /api/swipe {left} → INSERT Swipes, swipes_left−1.
  • • Вправо — {right}. Проверка взаимного лайка → INSERT Matches → модалка «💕 Совпадение!».
  • • Суперлайк ⭐ — свайп вверх. Цель получает пуш и всплывает первой в Симпатиях.
  • • swipes_left=0 → Paywall (blur + таймер до 00:00 МСК + «Стать Купидоном»).

Rewind ⏪

Только Premium/Kupidon. Free → 403 → Paywall «Случайно упустил судьбу? Верни с Premium». Платник: удаляем последний left за 24ч, swipes_left+1, карточка возвращается анимацией.

5. Симпатии & Врывы (Force Chat)

GET /api/sympathies → те, кто лайкнул тебя без существующего матча. Фото открыты у всех тарифов — блюра нет принципиально. Написать без взаимного свайпа = врыв.

Free

Видит анкеты. Клик → «Написать сейчас за 50 ⭐ или жди в ленте». Кнопка [Оплатить 50 Stars].

Premium — 3 врыва/сут

Клик → «Осталось {X} врывов. Войти сейчас?» → POST /api/force_chat: force_chats_used+1, Match с is_force_chat=true → редирект в чат.

Kupidon — 15 врывов/сут

То же, 15 врывов. Лимит исчерпан → докупить за Stars (100 ⭐ = 3 врыва).

«Кто смотрел тебя» new

Отдельная секция просмотров профиля. У Free — счётчик и силуэты, у Premium/Kupidon — открыто. Сильный триггер апгрейда.

6. Чаты, Безопасность & Согласие

Блокировка TG-контакта

  • • Кнопка Telegram в шапке серая/disabled по умолчанию.
  • • Счётчик «Откроется через {N} сообщений», обновляется через WS.
  • • Free: N=15. Premium: N=5. Kupidon: 0 (сразу, лимит 3 раза/сут).
  • • messages_count растёт при сообщении от обоих (защита от спама «привет»×15 в одну сторону: учёт по уникальным от каждого).

Запросить ТГ — мгновенное согласие

  • 1. Юзер 1 жмёт «Запросить ТГ» → POST /api/chat/request_tg.
  • 2. Юзеру 2 — системный пузырь ConsentBubble: «Собеседник хочет в Telegram» + [Отклонить] [Одобрить].
  • 3. Одобрить → tg_unlocked обоих = true, кнопка синеет мгновенно, лимит сообщений аннулируется.
  • 4. Отклонить → «Обмен отклонён», повтор через 24ч.

Ледоколы

messages_count=0 → 3 кнопки-подсказки над клавиатурой (GET /api/icebreakers, рандом из пула). Клик = текст уходит как обычное сообщение.

WebSocket-протокол

  • • WS /ws/chat/{match_id}?token=JWT.
  • • События: message_sent, message_read, typing, tg_consent_request/approved/declined, match_created, media_burned.
  • • Разрыв → Long Polling fallback каждые 3 сек.

7. Trust & Safety new

Без этого слоя дейтинг умирает от ботов, накруток и токсичности. Всё работает тихо: добросовестный юзер не замечает, абьюзер — упирается в стены.

Репорты

Кнопка ⚠️ в карточке и чате. POST /api/report {target, reason}. Причины: фейк, спам/реклама, оскорбления, NSFW, несовершеннолетний, мошенничество. 3+ репорта от разных → авто needs_review + temp-shadowban до проверки.

Блок

POST /api/block. Заблокированный исчезает из ленты обоих навсегда (двусторонне), матч/чат скрывается. INSERT в Blocks, фильтр NOT EXISTS в /api/feed.

AI-модерация медиа

Каждое фото/видео → NSFW-классификатор. confidence ≥ 0.85 → автоблок медиа + флаг модератору. В 18+ режиме порог жёстче. Аватарки без лица — флаг «не селфи».

Шадоубан

is_shadowbanned=true: юзер видит ленту и думает, что всё ок, но его лайки не пишутся для других и он не показывается (WHERE is_shadowbanned=false). Идеально против спам-ботов — они не понимают, что отключены.

Rate-limit & анти-фрод

Лимиты свайпов уже есть, плюс: антифлуд на сообщения, троттлинг регистраций с одного устройства, trust_score (возраст аккаунта, верификация, поведение). Низкий trust → меньше охват + капча-жест.

Забота о пользователе

В чате — ненавязчивая подсказка о безопасных встречах. Сообщения с запросом денег/крипты подсвечиваются как «возможный скам». Лёгкий путь «Пожаловаться и заблокировать» в один тап.

Ребилд — другая цель, без эротики

8. Комната 18+

Это не место для интимных фото. Это честное пространство для взрослых с открытыми намерениями: свидание без обязательств, флирт без серьёза, «один вечер», открытые отношения. Контент остаётся приличным — отличается только пул людей и теги.

Нюды, порно, интим — запрещены на уровне кода и AI-модерации (порог жёстче обычного). Цель режима — совпадение намерений, а не обмен голым телом.

Вход через верификацию

Тумблер 18+ доступен только verified-юзерам + чекбокс «Мне есть 18 и я хочу в 18+ пул». Тема Mini App уходит в тёмно-бордовую через React Context. Барьер от подростков.

Параллельный пул

is_18_mode_active=true → анкета исчезает из обычной ленты. Видишь только тех, у кого тот же флаг. Пулы полностью изолированы: GET /api/feed WHERE is_18_mode_active = me.is_18_mode_active.

Теги намерений (is_18_only)

Отдельный пул тегов про цель, не про тело. Создаётся командой /tags18:

🎲 Без обязательств 🌙 Один вечер 💫 Открытые отн. 🎭 Анонимно

Жёсткая модерация

Медиа → NSFW-классификатор с пониженным порогом. Интим → автобан + флаг. Ссылки/контакты в чате до анлока ТГ → regex-блок.

Исчезающие медиа

Match.is_18_room=true → Скрепка меняется на 🔥, фото = is_disappearing. У получателя блюр, клик = 1 просмотр, закрыл → burn_media → удаление с S3 навсегда. Скриншот-детект Telegram уведомляет отправителя.

Выход в один тап

Выключил тумблер → мгновенно возвращаешься в обычный пул, тема светлеет. 18+ матчи остаются в архиве чатов, но новые анкеты — только из белого пула.

Полный ребилд — анонимная интрига

9. Режим «Олигарх»

Было: купить TG-username за 1000 Stars. Стало: анонимный приоритетный сигнал + раскрытие только по взаимности.

Олигарх не покупает контакт грубой силой и никогда не получает юзернейм без согласия. Он шлёт анонимный VIP-сигнал. Девушка сама решает — раскрыть себя или нет. Всё держится на взаимности — это честно и не абьюзно.

Стелс-анкета

is_stealth_mode=true → анкета Олигарха невидима в общей ленте (WHERE is_stealth_mode=false у всех). Он сам выбирает, кому лайкнуть. Имя и фото скрыты до раскрытия.

Приоритетный VIP-сигнал

Олигарх лайкает → цель получает пуш «👑 VIP проявил интерес». В Симпатиях — карточка с размытым силуэтом и золотым бейджем, всплывает самой первой. Имя/фото скрыты.

INSERT INTO vip_notifications(target_id, swipe_id, is_revealed=false). Пуш bot.send_message(type='vip_like').

Раскрытие = взаимный лайк

Девушка жмёт «Раскрыть» (=лайк) → Match с is_oligarch_reveal=true, vip_notifications.is_revealed=true. Только теперь оба видят друг друга и открывается чат. ТГ-контакт — по обычной логике согласия. Отказ → сигнал тихо исчезает.

Анонимная записка

К сигналу можно прикрепить текст до 100 символов: «👑 Влиятельный незнакомец: "Хочу познакомиться"». Хранится в Swipes.vip_message, модерируется. Цена — 500 Stars за усиленный сигнал.

Анти-спам сигналами new

Чтобы Олигарх не заваливал всех подряд: лимит VIP-сигналов в сутки + кулдаун. Массовые лайки без раскрытий снижают доставляемость следующих сигналов (репутация). Анти-Олигарх щит у девушки делает её полностью невидимой для него.

10. Анти-Олигарх Щит

Полный контроль над тем, кто тебя видит. Один понятный тумблер — никаких запутанных уровней. Ни один Олигарх не достучится без твоего желания. Сильнейший маркетинговый аргумент для женской аудитории.

Что делает щит

  • • is_anti_oligarch=true → анкета полностью исчезает из стелс-ленты Олигархов.
  • • SQL Олигарха: WHERE u.is_anti_oligarch = false.
  • • Никаких VIP-пушей, никаких анонимных сигналов. Применяется мгновенно.
  • • Обычные парни видят тебя как всегда — щит блокирует только Олигархов.

Почему бесплатно?

Защита = доверие = больше активных девушек = живая платформа = больше платящих парней. Олигарх не теряет инструменты, он просто не может давить. Честная позиция продукта, которая монетизируется косвенно — через рост женской базы.

Маркетинг

«У нас VIP не может тебя купить. Только взаимность». Это месседж для женской аудитории и противовес репутации «дейтинга, где мужики с деньгами всё решают».

15. Ключевые алгоритмы

GET /api/feed — лента со скорингом

SELECT u.*, AGE(u.birth_date) AS age, -- скоринг совместимости (нормализуем 0..1 и взвешиваем) ( 0.30 * tier_boost(u.tier) + 0.25 * tag_overlap(:me_id, u.id) -- доля общих тегов + 0.20 * recency(u.last_active_at) -- свежесть активности + 0.10 * (u.profile_score / 100.0) + 0.10 * geo_proximity(:me_lat,:me_lng,u.lat,u.lng) + 0.05 * random() -- anti-staleness ) AS rank_score FROM users u WHERE (:search_gender = 'any' OR u.gender = :search_gender) AND u.is_stealth_mode = false AND u.is_shadowbanned = false AND u.is_banned = false AND u.id != :me_id AND u.is_18_mode_active = :me_18 -- изоляция пулов AND (:me_verified_only = false OR u.is_verified = true) AND (:me_is_oligarch = false OR u.is_anti_oligarch = false) -- щит AND NOT EXISTS (SELECT 1 FROM swipes s WHERE s.actor_id = :me_id AND s.target_id = u.id) AND NOT EXISTS (SELECT 1 FROM blocks b WHERE (b.blocker_id=:me_id AND b.blocked_id=u.id) OR (b.blocker_id=u.id AND b.blocked_id=:me_id)) AND ST_DWithin(geo(u), geo(:me), :radius_m) -- 50k→100k→200k при нехватке ORDER BY rank_score DESC LIMIT 15;

def handle_swipe(me, target_id, action_type, vip_message=None): # 1. Лимиты if me.swipes_left <= 0: raise HTTP(429, "Limit exceeded") if action_type == 'superlike' and me.superlikes_left <= 0: raise HTTP(402, "Buy superlikes") if me.is_oligarch_mode and me.vip_signals_used >= VIP_DAILY_CAP: raise HTTP(429, "VIP cap") # 2. Записать свайп (идемпотентно по UNIQUE(actor,target)) swipe = upsert_swipe(me.id, target_id, action_type, is_vip_like=me.is_oligarch_mode, vip_message=moderate(vip_message)) me.swipes_left -= 1 if action_type == 'superlike': me.superlikes_left -= 1 if action_type in ('right','superlike'): target = get_user(target_id) # 3a. Олигарх → анонимный приоритетный сигнал (если щит выключен) if me.is_oligarch_mode and not target.is_anti_oligarch: me.vip_signals_used += 1 insert_vip_notification(target.id, swipe.id) # is_revealed=false bot.send(target.tg_id, type='vip_like', text=vip_message) # 3b. Взаимный лайк? mutual = get_swipe(actor=target_id, target=me.id, in_=('right','superlike')) if mutual: match = create_match(me.id, target_id, is_18_room=(me.is_18_mode_active and target.is_18_mode_active), is_oligarch_reveal=me.is_oligarch_mode) if me.is_oligarch_mode: reveal_vip_notification(swipe.id) # только теперь раскрываем return {is_match: True, match_id: match.id} return {is_match: False}
