2. Онбординг & Профиль

1

Имя & Возраст

  • • Поле имени: только буквы/пробелы, 2–30 символов, trim(). Спецсимволы и цифры блокируются в реальном времени.
  • • Барабан даты рождения (iOS-стиль). Возраст вычисляется на лету.
  • • Возраст < 18 → кнопка «Далее» disabled + красная надпись «Доступ с 18 лет».
  • • В БД пишется birth_date (DATE) — возраст пересчитывается каждый год автоматически.
2

Я / Ищу

  • • «Я»: [Парень] [Девушка] — radio, обязательно.
  • • «Ищу»: [Парня] [Девушку] [Всех].
  • • Пишется в Users.gender + Users.search_gender.
  • • Жёсткий WHERE-фильтр в ленте.
3

Геолокация

  • • «Найти меня (GPS)» → разрешение браузера → координаты → POST /api/geo/resolve → PostGIS ищет ближайший город.
  • • Fallback: поиск города с автокомплитом.
  • • Smart Geo: анкеты кончились → автоматически расширяем радиус 50 → 100 → 200 км.
4

Медиа-слоты

  • • 5 слотов. Фото (до 5 МБ) = 1 слот, конвертируется в WebP. Первое фото — обязательно.
  • • Видео (до 10 сек, до 15 МБ) = 3 слота, UI визуально сливает квадраты.
  • • Видео при < 3 свободных слота → ошибка «Освободите 3 слота».
  • • Шкала «Горячести»: 1 фото=20%, каждый следующий слот +20%, видео +40%. Биография +10%.
5

Теги (до 5)

  • • GET /api/tags → облако кнопок с HEX-цветом обводки + эмодзи.
  • • Теги is_18_only показываются только в 18+ режиме.
  • • 5-й выбранный — остальные блокируются с анимацией «лимит».
  • • Сохраняется в User_Tags.
6

О себе + Финал

  • • Textarea до 150 символов со счётчиком. Необязательно, но даёт +10% к «горячести».
  • • Автомодерация: стоп-слова, ссылки, регулярки на бэкенде.
  • • После 6 шагов → INSERT INTO users → JWT-токен → вход в ленту.
  • • Редактирование в любое время из раздела Профиль.

3. Лента & Свайпы

Кэш-пачка

GET /api/feed возвращает 15 анкет и кэширует в Redis (TTL 5 мин). Фронтенд хранит локально. Новый запрос при ≤ 3 оставшихся. Устраняет задержки при каждом свайпе.

Свайп влево (Пропуск)

  • • Карточка улетает влево (Framer Motion, spring).
  • • POST /api/swipe {type:"left"} → INSERT INTO Swipes.
  • • swipes_left -= 1. При = 0 → Paywall.

Свайп вправо (Лайк)

  • • POST /api/swipe {type:"right"} → INSERT INTO Swipes. swipes_left -= 1.
  • • Проверка: SELECT * FROM Swipes WHERE actor_id=target AND target_id=me AND action_type='right'.
  • • Если взаимный → INSERT INTO Matches → is_match: true.
  • • Фронтенд: полноэкранная модалка «💕 СОВПАДЕНИЕ!» с аватарками + кнопка «Написать».

Суперлайк ⭐

  • • Свайп вверх или кнопка ⭐.
  • • Free: 0 бесплатных, купить за Stars. Premium/Kupidon: 5/сут бесплатно.
  • • Цель получает push «⭐ Кто-то поставил тебе суперлайк!» — карточка всплывает первой в Симпатиях.

Rewind ⏪

  • • Только Premium/Kupidon. POST /api/rewind.
  • • Free → HTTP 403 → Paywall «Случайно упустил судьбу? Верни с Premium!».
  • • Premium/Kupidon: удаляем последний action_type='left' за 24ч, swipes_left += 1, карточка возвращается анимацией.

Paywall

  • • swipes_left = 0 → компонент блокируется, backdrop-blur.
  • • Таймер до 00:00 МСК. Кнопка «Стать Купидоном 👑» с переливающейся анимацией.
  • • После покупки → swipes_left обновляется мгновенно через WebSocket-событие.

5. Чаты, Безопасность и Согласие

Блокировка TG-контакта

  • • Кнопка Telegram в шапке — серая, disabled по умолчанию.
  • • Счётчик «Откроется через {N} сообщений», обновляется через WS.
  • • Free: N=15. Premium: N=5. Kupidon: 0 (сразу, лимит 3 раза/сут).
  • • messages_count инкрементируется при каждом сообщении от обоих юзеров.

Запросить ТГ (Мгновенное Согласие)

  • 1. Юзер 1 нажимает «Запросить ТГ» → POST /api/chat/request_tg.
  • 2. Юзеру 2 появляется системный пузырь ConsentBubble: «Собеседник хочет в Telegram» + [Отклонить] [Одобрить].
  • 3. Одобрить → POST /api/chat/approve_tg → tg_unlocked обоих = true.
  • 4. Кнопка Telegram у обоих мгновенно синяя. messages_count-лимит аннулируется.
  • 5. Отклонить → системное «Обмен отклонён». Повторный запрос через 24ч.

Ледоколы (Icebreakers)

При messages_count=0 — 3 кнопки-подсказки над клавиатурой. GET /api/icebreakers → рандомные из пула. Клик = текст отправляется как обычное сообщение.

WebSocket протокол

  • • WS /ws/chat/{match_id}?token=JWT
  • • События: message_sent, message_read, typing_start, typing_stop, tg_consent_request, tg_consent_approved, tg_consent_declined, match_created.
  • • Разрыв → Long Polling fallback каждые 3 сек.

6. Комната 18+

Это не место для интимных фото. Это честное пространство для взрослых с открытыми намерениями: FWB, свидание без обязательств, открытые отношения, флирт без серьёза. Весь контент остаётся приличным — разница только в пуле людей и тегах.

Нюды, порно, интимные фото — запрещены на уровне кода и AI-модерации.

Активация

Скрытый тумблер в Настройках (появляется после верификации 18+). Чекбокс «Я совершеннолетний(яя) и хочу перейти в 18+ пул». Тема Mini App меняется на тёмно-бордовую через React Context.

Параллельный пул

is_18_mode_active=true → анкета исчезает из обычной ленты полностью. Видишь только тех, у кого тот же флаг. GET /api/feed WHERE is_18_mode_active = me.is_18_mode_active. Пулы полностью изолированы.

Теги 18+ (is_18_only)

Отдельный пул тегов, создаётся в админке командой /tags18. Примеры:

🔥 FWB 🎲 Без обязательств 🌙 Один вечер 💫 Открытые отн. 🎭 Анонимно

Автомодерация контента

  • • Фото/видео → NudeNet classifier на бэкенде (AI). Детект интимного контента.
  • • Если модель confident ≥ 0.85 → автобан медиа + уведомление модератору.
  • • Ссылки в чате → regex-блок до разблокировки TG.
  • • Стоп-слова в биографии → автомодерация с флагом needs_review.

Исчезающие медиа в чатах

Match.is_18_room=true → кнопка Скрепки меняется на 🔥. Все фото автоматически is_disappearing=true. У получателя — блюр. Клик = просмотр 1 раз. Закрыл → POST /api/chat/burn_media → удаляется с S3 навсегда.

Лимиты в 18+

Free в 18+: жёсткий лимит 5 свайпов/сут (вместо 50). Это стимул к апгрейду. Premium/Kupidon: обычные лимиты.

7. Режим «Олигарх»

Раньше: купить TG-username напрямую за 1000 Stars. Теперь: анонимная интрига + приоритетный сигнал.

Олигарх не покупает контакт грубой силой. Он отправляет анонимный приоритетный сигнал. Девушка сама решает — раскрыть или нет. Всё держится на взаимности.

Стелс-режим

is_stealth_mode=true → анкета Олигарха невидима в общей ленте. SQL: WHERE is_stealth_mode=false для всех остальных. Олигарх-призрак — сам выбирает, кому лайкнуть.

Приоритетное VIP-уведомление

Олигарх лайкает → цель получает push «👑 VIP проявил интерес к вашей анкете». В Симпатиях — карточка с размытым силуэтом и золотым бейджем. Имя и фото скрыты. Уведомление всплывает самым первым.

INSERT INTO vip_notifications(target_id, swipe_id, is_revealed=false).
Пуш через bot.send_message() с type='vip_like_notification'.

Взаимный лайк → Раскрытие

Девушка нажимает «Раскрыть» (=лайк) → INSERT INTO Matches с is_oligarch_reveal=true, vip_notifications.is_revealed=true. Олигарх получает мгновенный TG-контакт. Девушка видит его настоящее фото и имя.

VIP Суперлайк + анонимное сообщение

Олигарх может добавить текст до 100 символов. Девушка видит: «👑 Влиятельный незнакомец: "Хочу познакомиться"». Стоимость: 500 Stars. Хранится в Swipes.vip_message.

8. Анти-Олигарх Щит

Полный контроль над тем, кто тебя видит. Ни один Олигарх не достучится без желания. Это сильнейший маркетинговый аргумент для привлечения женской аудитории.

Уровень 1 — Щит (полная невидимость)

  • • is_anti_oligarch = true → анкета исчезает из стелс-ленты Олигархов.
  • • SQL для Олигарха: WHERE u.is_anti_oligarch = false.
  • • Никаких VIP-пушей. Никаких анонимных лайков. Применяется мгновенно.

Уровень 2 — Фильтр уведомлений

Щит выключен, но VIP-пуши не приходят. Анкета видима, лайк Олигарха просто попадает в симпатии без золотого бейджа — как обычный лайк. Анкета отображается в ленте Олигарха, но без спецэффектов.

Почему бесплатно?

Защита = доверие = больше активных девушек = живая платформа. Олигарх не теряет инструменты — он просто не может давить. Всё держится на взаимности. Честная позиция продукта.

10. God Mode (Бот-Админка)

Только для ADMIN_IDS. Все команды прямо в Telegram — никаких веб-панелей. Каждый handler: if message.from_user.id in ADMIN_IDS.

/user {id_или_@username}

SELECT юзера → стата (тариф, Stars, репорты, бан). Inline-клавиатура:

Теневой бан: юзер видит анкеты, думает что всё ок. Его лайки не пишутся в БД. В ленте других WHERE is_shadowbanned=false.
/tags Название | HEX | Эмодзи

INSERT admin_tags с is_18_only=false.

/tags18 Название | HEX | Эмодзи

То же, но is_18_only=true → только в 18+ пуле.

/economy param value

UPDATE config. Пример: /economy free_swipes 30. Без деплоя.

/stats

DAU/MAU, свайпов/24ч, новых регов, конверсия Free→Premium, активных Олигархов.

/ban {id} [причина]

12. Ключевые Алгоритмы

GET /api/feed — Алгоритм ленты

SELECT u.*, (DATE_PART('year', AGE(u.birth_date)))::INT AS age FROM users u WHERE (me.search_gender = 'any' OR u.gender = me.search_gender) AND u.is_stealth_mode = false AND u.is_shadowbanned = false AND u.is_banned = false AND u.id != me.id AND NOT EXISTS ( SELECT 1 FROM swipes s WHERE s.actor_id = me.id AND s.target_id = u.id ) AND u.is_18_mode_active = me.is_18_mode_active AND (me.is_oligarch_mode = false OR u.is_anti_oligarch = false) AND ST_DWithin( ST_MakePoint(u.lng, u.lat)::geography, ST_MakePoint(me.lng, me.lat)::geography, :radius_m -- 50000 → 100000 → 200000 при нехватке анкет ) ORDER BY CASE u.tier WHEN 'kupidon' THEN 1 WHEN 'premium' THEN 2 ELSE 3 END, CASE WHEN u.created_at > NOW() - INTERVAL '2 hours' THEN 0 ELSE 1 END, RANDOM() LIMIT 15;

def handle_swipe(me, target_id, action_type, vip_message=None): # 1. Проверить лимиты if me.swipes_left <= 0: raise HTTPException(429, "Limit exceeded") if action_type == 'superlike' and me.superlikes_left <= 0: raise HTTPException(402, "Buy superlikes") # 2. Записать свайп swipe = INSERT INTO swipes( actor_id=me.id, target_id, action_type, is_vip_like=me.is_oligarch_mode, vip_message=vip_message ) # 3. Списать лимиты UPDATE users SET swipes_left = swipes_left - 1 WHERE id = me.id if action_type == 'superlike': UPDATE users SET superlikes_left = superlikes_left - 1 WHERE id = me.id # 4. Обработка лайка/суперлайка if action_type in ('right', 'superlike'): # 4a. Олигарх → приоритетное VIP-уведомление if me.is_oligarch_mode: target = SELECT * FROM users WHERE id = target_id if not target.is_anti_oligarch: # щит не активен INSERT INTO vip_notifications(target_id, swipe_id=swipe.id) bot.send_message(target_id, type='vip_like', text='👑 VIP проявил интерес', vip_message=vip_message) # 4b. Проверить взаимный лайк mutual = SELECT * FROM swipes WHERE actor_id = target_id AND target_id = me.id AND action_type IN ('right', 'superlike') if mutual: match = INSERT INTO matches( user1_id=me.id, user2_id=target_id, is_18_room=(me.is_18_mode_active AND target.is_18_mode_active), is_oligarch_reveal=me.is_oligarch_mode ) if me.is_oligarch_mode: UPDATE vip_notifications SET is_revealed=true WHERE swipe_id=swipe.id return {is_match: true, match_id: match.id} return {is_match: false}
